What is Javascript?

JavaScript is a high-level, interpreted, object-oriented programming language, primarily used in web development, intended to add interaction potential and dynamism to web pages.

It provides a wide range of functionality, such as DOM (Document Object Model) manipulation to interact with elements of a web page, event handling, communication with servers via AJAX, creating animations, form validation, and much more. It is a flexible and dynamic language that allows developers to create interactive, content-rich experiences.

What is Javascript Asynchronous?

In JavaScript, async simplifies handling asynchronous operations by allowing the use of await within asynchronous functions, making it easier to wait for promises to resolve without needing to chain multiple .then() blocks together. This improves code readability and prevents "callback hell", allowing asynchronous code to resemble synchronous code and be easier to follow. It also facilitates error handling via try and catch, making exception handling in the asynchronous flow more straightforward and manageable. 

                    
                        // Asynchronous function that returns a message
                        async function sayHello() {
                            return '¡Hello, World!';
                        }

                        // Function that uses await to wait for the result of "sayHello"
                        async function greet() {
                            const message = await sayHello(); // We wait for "sayHello" to finish
                            console.log(message); // We print the message
                        }

                        // We call the asynchronous function
                        greet();

What is Callback?

A callback function is one that is passed as an argument to another function to be "called back" at a later time. A function that accepts other functions as arguments is called a High-Order function, and contains the logic to determine when the callback function is executed. It is the combination of these two that allows us to extend our functionality. 

                    
                        function createQuote(quote, callback) {
                            let myQuote = "As I always say, " + quote;
                            callback(myQuote); // 2
                        }
                        
                        function logQuote(quote) {
                            console.log(quote);
                        }
                        
                        createQuote("Eat your vegetables!", logQuote); // 1
                        
                        // Output to console:
                        // As I always say, eat your vegetables!

What is Promise?

A Promise (Promise) is a proxy for a value that is not necessarily known when the promise is created. It allows you to associate handlers with the eventual success value or failure reason of an asynchronous action. This allows asynchronous methods to return values ​​like synchronous methods: instead of immediately returning the final value, the asynchronous method returns a promise to provide the value at some point in the future.

A Promise is in one of these states:

  • pending - Initial state, neither fulfilled nor rejected.
  • fulfilled - Meaning the operation completed successfully.
  • rejected - Meaning the operation failed.

A pending promise can be fulfilled with a value or rejected with a reason (error). When either of these occurs, the associated handlers queued by the then method of a promise are called. If the promise has already been fulfilled or rejected when a corresponding handler is attached, the handler will be called, so there is no race condition between the completion of an asynchronous operation and the attachment of its handlers. 

                    
                        // Function that returns a promise
                        function checkEvenNumber(number) {
                            return new Promise((resolve, reject) => {
                                setTimeout(() => {
                                    if (number % 2 === 0) {
                                        resolve('The number is even.');
                                    } else {
                                        reject('The number is odd.');
                                    }
                                }, 2000); // Wait for 2 seconds
                            });
                        }
                
                        // Using the promise
                        checkEvenNumber(4)
                            .then((result => {
                                console.log(result); // If the number is even, prints "The number is even."
                                return 'Next operation after checking.';
                            })
                            .then(next => console.log(next)) // Prints "Next operation after checking."
                            .catch(error => console.log(error)); // If the number is odd, prints "The number is odd."

What is Async/Await?

Async/Await is a feature in JavaScript that simplifies asynchronous programming by allowing you to write asynchronous code that looks and behaves like synchronous code. An async function always returns a Promise, and within this function, the await keyword can be used to pause execution until a Promise is resolved or rejected. This makes asynchronous code easier to read and manage, as it eliminates the need for complex Promise chaining and simplifies error handling with try/catch blocks. 

                    
                        // Async function to fetch data from an API
                        async function fetchData() {
                            const url = 'https://jsonplaceholder.typicode.com/posts'; // API URL
                
                            try {
                                let response = await fetch(url);
                
                                if (!response.ok) {
                                    throw new Error('Network response was not ok');
                                }
                
                                let data = await response.json();
                
                                console.log('Data received:', data);
                            } catch (error) {
                                console.error('There was a problem with the request:', error);
                            }
                        }
                
                        // Call the async function
                        fetchData();

What is Event Loop?

The Event Loop is a fundamental part of JavaScript's concurrency model, enabling non-blocking, asynchronous operations in the language. It allows JavaScript to perform tasks like handling events, executing callbacks, and processing asynchronous operations without blocking the main thread.

How It Works

  1. Call Stack: The call stack keeps track of function calls. When a function is invoked, it is added to the call stack. Once it finishes executing, it is removed from the stack.
  2. Web APIs: JavaScript runtime environments like browsers or Node.js provide Web APIs (e.g., setTimeout, fetch). These APIs handle tasks like timers and network requests asynchronously.
  3. Callback Queue (Task Queue): When Web APIs complete their tasks, they push callbacks (functions to be executed) into the callback queue.
  4. Event Loop: The event loop continuously checks the call stack and the callback queue. If the call stack is empty, it takes the first callback from the callback queue and pushes it onto the call stack for execution.
                    
                        console.log('Start');
                        
                        setTimeout(() => {
                            console.log('Timeout callback');
                        }, 2000); // Executes after 2 seconds
                        
                        console.log('End');
